# 微网数电精简讲法

## 先说结论
这版你不要当“作品交付”去讲，也不要顺着整页慢慢念。  
你就把它当成一个 `我如果来做这个岗位，我会怎么拆问题` 的展示。

## 最自然的开场
“我其实不是想证明我对新能源懂得有多深，我主要是想展示一下，如果让我做这个岗位，我会怎么先梳理补能赛道，再把公开信息整理成一个能支持判断的框架。”

## 一分钟半版本
“我会先对补能赛道做一个简短梳理，再结合微网数电公开业务信息，把它按几个不同场景来看：两轮换电、聚合超充、BaaS、光储充。因为这几类场景的收入驱动、资产结构和风险点都不一样，所以不能混在一起看。  

然后如果真到商业分析，我会先过五道门，不会一上来就写很复杂的模型。五道门就是需求密度、接电与站址、资产经济性、运营可靠性，还有合规和伙伴协同。  

再往下一步，如果落到具体项目，比如一个城市快充站点，我会先做轻量初筛，重点抓几个最影响结论的变量，比如服务量、功率利用率、接电条件、租金和在线率，然后给一个推进、暂缓还是放弃的判断。  

最后我会把这个过程做成可复用的工作流，而不是一次性文档。也就是把资料收集、字段标准化、决策表和更新机制串起来，用 Excel 和 Python 让后面的筛选和测算更快一点。”

## 两分钟版本
“这页我主要想说明一件事，就是这个岗位如果让我来做，我不会停在资料整理层面，我会尽量把它转成判断框架。  

我第一步会先对补能赛道做一个简短梳理，再结合微网数电公开看下来的业务方向，把它分成换电、聚合超充、BaaS 和光储充几个场景。拆开以后，每个场景的关键变量就完全不一样。比如换电更偏运营效率，超充更偏单位经济和接电条件，BaaS 更偏资产管理，光储充更偏系统协同。  

第二步就是筛机会。我不会因为市场还在涨就默认项目值得做，而是会先过五道门：需求够不够密、站址和接电能不能落、资产经济性是不是健康、运营能不能稳定、合规和外部协同是不是跟得上。  

第三步才是项目初筛。比如一个城市快充站点，我会先抓最影响结论的变量，像服务量、功率利用率、租金、电价、在线率这些，先给一个推进、暂缓还是放弃的判断。  

最后我觉得这个岗位真正能产生价值的地方，不是做出一份研究，而是把资料收集、字段标准化和判断模板沉淀下来，后面可以持续支持项目筛选和测算。”  

## 现场怎么点
1. 先开 `live-walkthrough.html`
2. 只讲这几个区块：
   - `总览`
   - `四个场景`
   - `五道门`
   - `一个示例`
   - `我的做法`
3. 不要讲完整案例页
4. 不要主动讲来源页
5. 不要逐条念 bullet

## 你讲的时候要记住的三句
1. “我先拆场景，不先讲大盘。”
2. “我先筛机会，不先堆模型。”
3. “我会把分析做成后面还能复用的流程。”

## 最后收口
“所以这页我想展示的不是我已经把这个行业研究透了，而是如果让我来做这个岗位，我会怎么把公开信息快速整理成可判断、可推进、后面也能持续复用的分析框架。”
