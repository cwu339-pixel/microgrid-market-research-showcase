A1

Research Basis

研究方法与资料范围

IEA｜Global EV Outlook 2025

旗舰研究的标准骨架

IEA 的 Global EV Outlook 2025 把内容拆成市场、产业、affordability、charging、batteries、outlook。这个骨架说明，成熟研究不是一堆资料，而是按 市场 → 供给 → 基础设施 → 展望 去组织。

McKinsey｜Charging + Consumer

按场景拆，不按抽象行业讲

McKinsey 在 EV charging 和中国汽车消费者研究里反复强调： 城市、高速、家庭、工作场景的需求结构不同；消费者对 charging experience 的不满会直接影响选择。也就是研究不能只讲总量，要讲场景。

Deloitte｜Infrastructure Ecosystem

基础设施不是单设备问题

Deloitte 关于 fleet charging 的公开报告强调的不只是建站，而是 charging infrastructure + commercial ecosystem 的组合决策。这个思路对微网数电也适用：供应商、金融、配网、运维要一起看。

中国官方政策文件

把监管约束直接放进商业模型

发改委、能源局、财政部、国家消防救援局这些文件告诉你： 充换电行业已经进入 高质量建设 + 储能协同 + 合规运营 阶段，合规和接电效率都不是附属变量，而是前置条件。

政策层

六部门“三年倍增”行动方案把重点写成 城市快充、大功率、高速、乡镇、V2G，说明行业不再只是“补数量”。

市场层

IEA 公开分析显示，中国仍是 fast charging 增长的全球主导者，场景密度和充电体验仍然是竞争变量。

监管层

县域试点、共享换电工作指南、获得电力意见，都把 可用率、接电时效、安全和平台留痕 直接推成经营问题。

研究含义： 商业分析不能只回答“这个市场大不大”，还要回答“在什么场景、以什么功率结构、在什么合规成本下，这个项目还能不能成立”。

02

Company Scenes

同一家公司，四套商业模型

Scene A

两轮换电

  • 看站点密度、机柜可用率、电池周转
  • 看价格公示、消防合规、实时监测
  • 主要问题：高频运营下的库存与周转效率

Scene B

聚合超充 / 城市快充

  • 看接电、功率结构、峰谷价差和在线率
  • 看站点回本周期与单站服务量
  • 主要问题：功率利用率能否覆盖资本开支

Scene C

BaaS 电池银行

  • 看 SOH、寿命、残值、质保成本
  • 看不同客户场景的使用差异
  • 主要问题：寿命、残值与质保成本是否可控

Scene D

光储充与数字调度

  • 看储能是否改善成本曲线
  • 看调度是否带来经营杠杆
  • 主要问题：协同收益能否显著改善单位经济

核心判断： 同一家公司，不同场景，不同模型。先分层，再测算，才不会把“大行业逻辑正确”误当成“单项目逻辑成立”。

03

Opportunity Screening

先过五道门，再决定是否投入销售资源

01

需求密度

客群在哪、频次够不够、竞争是不是已经太挤。

02

接电与站址

配网承载、接电时效、物业条件、消防复杂度。

03

资产经济性

设备投入、电池折旧、寿命、残值与维护成本。

04

运营可靠性

可用率、成功率、故障率、库存周转、人效。

05

合规与伙伴

政策适配、价格公示、数据留痕、供应链与金融协同。

筛选输出应形成： 结论 + 关键假设 + 缺失数据 + 下一步验证动作 ，而不是“资料还没齐，所以先无法判断”。

04

Decision Template

项目初筛示例：先看单位经济，再看扩张价值

Illustrative Case

城市快充站点初筛

在未取得内部经营数据前，初筛重点不是把模型做复杂，而是先确认几个足以改变结论的关键变量是否成立。

先看输入条件

  • 站址所在区域的车流密度与停留时长
  • 接电时效、配网容量与物业条件
  • 峰谷价差、租金、人力与运维约束

再看单位经济

  • 单站服务量能否覆盖固定成本
  • 功率利用率是否支撑设备投入
  • 在线率和故障率是否吞噬毛利

最后给决策

  • 推进：关键假设已满足，进入详细测算
  • 暂缓：需求成立，但接电或站址条件未满足
  • 放弃：单位经济无法成立，且缺乏结构性改善空间

决策输出： 商业分析的交付不应停留在研究摘要，而应明确给出 推进 / 暂缓 / 放弃 及其依据、缺口数据和下一步动作。

05

Data Organization

自动化的价值，是把一次判断变成可复用的决策系统

Step 1

Source Intake

政策、行业、竞品、公司公开资料和项目底表统一收口。

Step 2

Normalization

把来源、日期、口径、场景、变量名称做标准化。

Step 3

Decision Table

映射到机会筛选、项目测算、定价或商务判断模板。

Step 4

Weekly Update

用 Excel / Python 维持更新，不让每次研究都重头来。

Excel 模板 字段字典 来源留痕 敏感性分析 版本管理 自动刷新 复盘闭环

执行原则： Python / Excel / 简单流程自动化 的价值，不是炫工具，而是把一份研究变成可以复用的工作流。
