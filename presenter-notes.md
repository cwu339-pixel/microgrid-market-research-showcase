# 微网数电案例页讲解稿

## Talk Objective
让面试官记住两件事：
1. 我会先对补能赛道做公开信息梳理，再结合公司公开业务方向去看具体场景。
2. 我能把公开信息整理成结构化判断，而不是只做一次性研究材料。

## 2-Minute Version
先说明一下，这页不是在猜公司内部数据，而是我用公开政策、公司披露和行业资料，先做一版补能赛道梳理，再结合微网数电公开业务信息看可能涉及的业务场景。

我的核心判断是，微网数电公开看下来不能被当成一个单一业务去谈。至少可以分成换电、聚合超充、BaaS 和光储充这几类场景，它们的需求频次、资本占用、资产寿命和合规约束都不一样，所以要分开看。

如果我来做这个岗位，我会先用五个维度筛机会：需求密度、接电与站址、资产经济性、运营可靠性、合规与伙伴协同。只有几个关键门槛同时成立，项目才值得继续推进。

再往下一步，我不会只停留在研究结论，而会把这些判断沉淀成可复用的 decision table。也就是把资料收集、字段标准化、机会筛选和后续测算串起来，用 Excel 和 Python 做成持续更新的工作流。

所以这页最想展示的不是我知道多少行业名词，而是我做商业分析时会怎么把公开信息变成判断框架和实际动作。

## 3-Minute Version
先说明一下，这页不是在猜公司内部数据，而是我基于公开政策、公司披露和行业研究，先对补能赛道做一个简短梳理，再结合微网数电公开业务信息，看这个岗位如果要做市场研究、机会筛选和数据整理，大概会怎么展开。

我先看行业信号。补能基础设施总量还在扩张，但判断重点已经不只是市场够不够大，而是单站经济性能不能成立，尤其是接电条件、功率利用率、资产寿命和合规成本。

在这个前提下，我会把微网数电拆成四类分析场景：两轮换电、聚合超充、BaaS 和光储充。因为这四类场景的收入驱动、成本结构和风险变量完全不同，所以不能混在一起做测算。

接着就是机会筛选。我会先过五道门：需求密度、接电与站址、资产经济性、运营可靠性、合规与伙伴协同。筛选输出不该是一份研究摘要，而应该是推进、暂缓或放弃，以及关键假设和缺口数据。

如果往项目层再走一步，我会先做一个轻量的初筛模型。重点不是一开始就建一个很复杂的财务模型，而是先确认几个能改变结论的变量，比如单站服务量、功率利用率、租金和运维约束、设备在线率这些。

最后，分析不能停在结论。我会把信息收集、字段标准化、决策表和周更机制串起来，用 Excel 和 Python 做成一个可以复用的工作流。这样每来一个新机会，不是重做一遍，而是在现有框架里快速更新判断。

## Likely Follow-ups
### 你不是能源专业，怎么补这个 gap？
我的做法不是先装作很懂全部技术，而是先把商业问题拆清楚，再把最关键的行业变量补齐。这个岗位更看重判断框架、信息整理、测算能力和协同能力，这几项我能更快产生价值。

### 为什么按四个场景拆？
因为换电、超充、BaaS、光储充的需求结构、资本占用、资产寿命和合规要求都不同。如果不先拆场景，后面所有测算都会失真。

### 你会怎么做初步项目测算？
先做轻量初筛，抓几个足以影响结论的变量，比如服务量、功率利用率、租金、在线率和运维成本。确认关键门槛成立后，再决定是否进入详细模型。

### Excel / Python 在这里具体怎么提效？
我会把资料收集、字段字典、决策表和敏感性分析模板标准化。Python 负责清洗和更新，Excel 负责快速测算和业务协作。

### 这页最想展示什么？
不是我知道多少行业名词，而是我能把公开信息整理成结构化判断，并进一步变成筛选机会和支持测算的工作流。
